PRIMARY COLOR: VIBRANT BLUE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Hex Code:           #0c54e7
  Light Mode OKLCH:   oklch(52.4% 0.21 265.5)
  Dark Mode OKLCH:    oklch(64% 0.22 265.5)

  Usage:  Buttons • Links • Primary Actions • Focus Rings • Form Elements

  Tailwind Classes:
    bg-primary                  → Blue background
    text-primary                → Blue text
    text-primary-foreground     → White text on blue
    bg-primary/50               → 50% opacity blue
    hover:bg-primary/90         → Hover state
    ring-primary                → Focus ring

OKLCH COLOR SPACE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

CSS Custom Properties (Use in CSS):
  --primary:                  oklch(52.4% 0.21 265.5)   [Light Mode]
  --primary:                  oklch(64% 0.22 265.5)     [Dark Mode]
  --secondary:                oklch(67.3% 0.157 155.8)  [Light Mode]
  --secondary:                oklch(75% 0.165 155.8)    [Dark Mode]
  --accent:                   oklch(71.5% 0.167 73.5)   [Light Mode]
  --accent:                   oklch(78% 0.175 73.5)     [Dark Mode]

Why OKLCH?
  ✓ Perceptually uniform color differences
  ✓ Accurate lightness representation (L component)
  ✓ Better for light/dark mode adjustments
  ✓ Future-proof with browser support

QUICK TEST CHECKLIST
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Before deploying color changes:
  ☐ Test in Light Mode
    ☐ Primary buttons visible and vibrant
    ☐ Green badges stand out
    ☐ Amber warnings are noticeable

  ☐ Test in Dark Mode
    ☐ All colors remain visible
    ☐ No color is too dim or washed out
    ☐ Contrast remains good

  ☐ Accessibility Checks
    ☐ Run contrast checker (minimum 4.5:1)
    ☐ Test with color blindness simulator
    ☐ Verify all text is readable

  ☐ Component Testing
    ☐ Buttons work as expected
    ☐ Badges display correctly
    ☐ Form validation colors show
    ☐ Status indicators are clear
